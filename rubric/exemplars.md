# Exemplars and recurring corrections

This file starts empty. When the same kind of editor correction shows up
3+ times across files in `lessons/`, add it here as a standing rule or a
good/bad example pair, so future generations get it right the first time.

## Format for each entry

### <short description of the pattern>
- **Bad:** <example of what Claude generated>
- **Good:** <example of the corrected version>
- **Why:** <what the correction is protecting against>
